# 📊 ESG 탄소 관리 시스템 상세 DB 스키마 설계서

본 설계는 데이터 유입부터 최종 리포팅까지의 5단계 프로세스를 기술적으로 뒷받침하며, 특히 **AI 에이전트의 판단 근거(신뢰도, 분석 의견)**를 기록하여 데이터의 투명성을 확보하는 데 중점을 두었습니다.

---

## 1. 기준 정보 및 자동 매핑 (1~2단계)
에이전트가 수집된 텍스트에서 키워드를 분석하여 배출원과 계수를 자동으로 연결할 때 참조하는 마스터 데이터입니다.

### `emission_factors` (배출 계수 마스터)
| 컬럼명 | 타입 | 제약조건 | 설명 |
| :--- | :--- | :--- | :--- |
| `factor_id` | INT | PK, AI | 계수 고유 식별자 |
| `category_scope` | VARCHAR(20) | NOT NULL | Scope 분류 (Scope 1, 2, 3) |
| `resource_name` | VARCHAR(100) | NOT NULL | 에너지원명 (예: LNG, 휘발유, 전력) |
| `keywords` | TEXT | - | **에이전트 매핑용 키워드 (JSON/CSV)** |
| `unit` | VARCHAR(20) | NOT NULL | 계수 기준 단위 (kWh, L, m3 등) |
| `ef_value` | DECIMAL(18,10)| NOT NULL | 탄소 배출 계수 ($EF$) |
| `gwp_value` | DECIMAL(18,10)| DEFAULT 1 | 지구 온난화 지수 ($GWP$) |
| `source_year` | INT | - | 계수 공표 연도 (관리용) |

---

## 2. 실적 데이터 및 AI 전처리 (1~3단계)
다양한 경로로 유입된 활동 데이터를 저장하고, 에이전트가 1차 가공(키워드 추출 및 매핑)한 결과를 기록합니다.

### `activity_data_logs` (활동 데이터 통합 로그)
| 컬럼명 | 타입 | 제약조건 | 설명 |
| :--- | :--- | :--- | :--- |
| `log_id` | BIGINT | PK, AI | 로그 고유 식별자 |
| `source_type` | VARCHAR(20) | - | 데이터 출처 (EXCEL, API, OCR 등) |
| `raw_input` | TEXT | - | **수집된 원문 텍스트 (AI 분석 대상)** |
| `facility_id` | INT | FK | 발생 사업장/시설 ID |
| `usage_value` | DECIMAL(18,4) | - | 에이전트가 추출한 사용량 수치 |
| `unit` | VARCHAR(20) | - | 추출된 단위 |
| `factor_id` | INT | FK | **에이전트가 자동 매핑한 계수 ID** |
| `mapping_score` | DECIMAL(3,2) | - | **매핑 신뢰도 지수 (0.00 ~ 1.00)** |
| `status` | VARCHAR(20) | - | 상태 (PENDING, VALIDATED, ERROR) |
| `created_at` | TIMESTAMP | DEFAULT NOW | 데이터 수집 일시 |

---

## 3. 이상치 탐지 및 검증 (3단계)
데이터의 신뢰성을 검증하고, 에이전트가 발견한 특이사항 및 피드백을 관리합니다.

### `validation_results` (이상치 검증 리포트)
| 컬럼명 | 타입 | 제약조건 | 설명 |
| :--- | :--- | :--- | :--- |
| `validation_id` | BIGINT | PK, AI | 검증 고유 ID |
| `log_id` | BIGINT | FK | 대상 활동 데이터 ID |
| `is_anomaly` | BOOLEAN | - | 이상치 판정 여부 |
| `anomaly_type` | VARCHAR(50) | - | 이상 유형 (STATISTICAL, LOGICAL 등) |
| `deviation_rate` | DECIMAL(7,2) | - | **평균 대비 이탈률 (%)** |
| `agent_comment` | TEXT | - | **에이전트의 분석 의견 및 질문** |
| `is_confirmed` | BOOLEAN | DEFAULT FALSE | 사용자 최종 승인 여부 |

---

## 4. 제품별 PCF 산정 및 배분 (4단계)
BOM(부품명세서) 구조를 반영하여 최종 제품별 탄소 배출량을 합산하고 관리합니다.

### `product_pcf_results` (제품별 탄소 성적표)
| 컬럼명 | 타입 | 제약조건 | 설명 |
| :--- | :--- | :--- | :--- |
| `pcf_id` | BIGINT | PK, AI | PCF 계산 결과 고유 ID |
| `product_code` | VARCHAR(50) | NOT NULL | 제품 식별 코드 |
| `target_period` | VARCHAR(7) | NOT NULL | 산정 기간 (예: 2026-05) |
| `total_pcf` | DECIMAL(18,6)| - | **최종 합산 배출량 (tCO2eq)** |
| `scope1_sum` | DECIMAL(18,6)| - | Scope 1 세부 합계 |
| `scope2_sum` | DECIMAL(18,6)| - | Scope 2 세부 합계 |
| `scope3_sum` | DECIMAL(18,6)| - | Scope 3 세부 합계 |
| `bom_snapshot` | JSON | - | **계산 당시의 BOM 구조 스냅샷** |

---

## 5. 분석 리포트 및 결과 출력 (5단계)
규제 대응 분석 결과와 에이전트가 생성한 요약 리포트를 관리합니다.

### `regulatory_reports` (규제/분석 리포트)
| 컬럼명 | 타입 | 제약조건 | 설명 |
| :--- | :--- | :--- | :--- |
| `report_id` | INT | PK, AI | 리포트 고유 ID |
| `pcf_id` | BIGINT | FK | 연결된 PCF 결과 ID |
| `report_type` | VARCHAR(20) | - | 리포트 종류 (CBAM, ESG_PUBLIC 등) |
| `estimated_tax` | DECIMAL(18,2) | - | **CBAM 예상 관세액** |
| `briefing_text` | TEXT | - | **에이전트 자동 생성 요약 브리핑** |
| `file_path` | VARCHAR(512) | - | 생성된 엑셀/PDF 파일 경로 |
| `generated_at` | TIMESTAMP | DEFAULT NOW | 리포트 생성 일시 |




# 🏭 알루미늄 제조 기업 PCF 산정을 위한 필요 데이터 목록

## 1. 기준 정보 (Master Data)
시스템의 계산 로직을 구성하는 기초 데이터입니다.
*   **BOM (Bill of Materials):** 알루미늄 빌렛(Billet) 또는 압출 제품 1톤을 생산하기 위해 투입되는 원재료 명세 (예: 알루미늄 잉곳 90%, 실리콘 2%, 마그네슘 1%, 알루미늄 스크랩 7%).
*   **공정 마스터:** 용해(Melting) -> 유지(Holding) -> 주조(Casting) -> 균질화(Homogenizing) 등 주요 공정 단계 정의.
*   **배출 계수 DB:** 
    *   **국가 표준 계수:** 전력(kWh), LNG($m^3$) 등 에너지원별 계수.
    *   **공급망 계수:** 매입하는 알루미늄 잉곳(Ingot)의 탄소 집약도(고순도 신재 vs 재생 알루미늄).

## 2. 활동 데이터 (Activity Data - Scope 1 & 2)
공장 내부에서 실제로 발생하는 에너지 사용량 데이터입니다.
*   **직접 배출 (Scope 1):**
    *   **용해로 연료:** LNG 또는 도시가스 사용량 (알루미늄은 용해 공정에서 가장 많은 연료를 소비함).
    *   **운송 장비:** 공장 내 지게차용 디젤, 법인 차량 휘발유 사용량.
*   **간접 배출 (Scope 2):**
    *   **전력 사용량:** 공장 전체 및 주요 설비별(압출기, 냉각 시스템 등) 전기 사용량 (고지서 데이터).

## 3. 공급망 데이터 (Upstream Data - Scope 3)
알루미늄 제조 원가의 대부분을 차지하는 원재료 관련 데이터입니다. (CBAM 대응 핵심)
*   **원재료 매입 내역:** 구매한 잉곳, 합금 첨가제 등의 중량 및 구매처 정보.
*   **내재 탄소량(Embedded Emissions):** 협력사로부터 제공받은 원재료의 탄소 배출 정보.
    *   *참고: 재생 스크랩(Scrap) 사용 비중이 높을수록 최종 PCF가 낮아지므로 스크랩 분류 데이터가 중요함.*
*   **물류 데이터:** 원재료 입고 시 발생하는 운송 수단 및 거리 정보.

## 4. 생산 및 운영 데이터 (Output & Operating)
탄소 배출량을 제품별로 쪼개기(배분) 위한 데이터입니다.
*   **제품별 생산 실적:** 특정 기간 동안 생산된 알루미늄 제품군별 총 중량.
*   **공정 가동 시간:** 설비별 가동 시간 (공통 에너지 사용량을 제품별로 배분하는 기준점).
*   **스크랩 및 폐기물:** 공정 중 발생하는 알루미늄 칩(Chip)이나 스크랩 발생량 및 재투입량.

## 5. AI 에이전트 학습용 특화 데이터
에이전트가 '이상치'를 탐지하거나 '자동 매핑'을 하기 위해 필요한 데이터입니다.
*   **과거 에너지 고지서 패턴:** 월별/계절별 정상 범위의 에너지 사용량 이력.
*   **협력사 명칭 및 품목 리스트:** 텍스트 분석 시 협력사 이름만 보고도 어떤 원재료인지 매핑하기 위한 학습 사전.
*   **글로벌 규제 가이드라인:** CBAM 산정 방식(Simple/Complex goods 분류) 등의 텍스트 데이터.
